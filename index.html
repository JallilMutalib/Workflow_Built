Workflow Architecture

Six nodes handle the full lifecycle — from scheduled trigger to published post and logged result. Both Facebook and Instagram get AI-tailored captions via a Switch node.

Step-by-Step Guide

How to Build This in n8n

Each node explained with exactly what to configure — so you can replicate or customise this workflow from scratch.

1
⏰ Schedule Trigger Trigger Node

This is the starting point of the workflow. It fires automatically at a set time every day without any manual action. In n8n, use the built-in Schedule Trigger node.

How to configure it
Add a Schedule Trigger node → set Mode to Cron → enter the expression 0 9 * * * for 9 AM daily (adjust to your timezone). No credentials needed. This node has no inputs — it just kicks everything off on time.
2
📋 Read Google Sheets Google Sheets Node

Pulls your content calendar from Google Sheets. Your sheet should have columns: Topic, Platform, Tone, Status. This node reads only rows where Status = "Pending".

How to configure it
Add a Google Sheets node → authenticate via OAuth2 → set Operation to Get Many Rows → paste your Sheet ID → set Filters: Status equals Pending. This returns all pending posts as individual items for the loop.
3
🔁 Loop Over Items SplitInBatches Node

Processes each pending post one by one instead of all at once. This prevents API rate-limit errors when publishing to Facebook and Instagram simultaneously.

How to configure it
Add a SplitInBatches node → set Batch Size to 1. Connect the "loop" output back to the AI Caption node, and the "done" output to the end (or a notification step). This ensures posts are processed sequentially, one at a time.
4
🤖 AI Caption Generator OpenAI Node

Sends the Topic and Platform fields to OpenAI GPT-4o. The AI writes a platform-appropriate caption — casual and emoji-friendly for Facebook, visually descriptive with hashtags for Instagram.

How to configure it
Add an OpenAI node → connect your API key → set Model to gpt-4o → set the prompt to: "Write a social media caption for {{ $json.Topic }} optimised for {{ $json.Platform }}. Tone: {{ $json.Tone }}." The output will be the AI-written caption stored in the response field.
5
🔀 Switch by Platform Switch Node

Reads the Platform field and routes each item to the correct publishing node. Facebook posts go one way, Instagram posts go another. Only two output paths are needed.

How to configure it
Add a Switch node → set Mode to Rules → Rule 1: {{ $json.Platform }} equals Facebook → Route 1. Rule 2: {{ $json.Platform }} equals Instagram → Route 2. Connect Route 1 to the Facebook node and Route 2 to the Instagram node.
6a
📘 Post to Facebook HTTP Request Node

Publishes the AI-generated caption to your Facebook Page using the Facebook Graph API. Requires a Facebook Page Access Token with pages_manage_posts permission.

How to configure it
Add an HTTP Request node → Method: POST → URL: https://graph.facebook.com/v19.0/YOUR_PAGE_ID/feed → Body Parameters: message = {{ $json.choices[0].message.content }}, access_token = your Page Token. The API returns a post ID on success.
6b
📸 Post to Instagram HTTP Request Node

Publishes to your Instagram Business Account via the Meta Graph API. Instagram requires a two-step process: first create a media container, then publish it.

How to configure it
Step A — POST to https://graph.facebook.com/v19.0/YOUR_IG_ID/media with caption and image_url fields. Get the creation_id from the response. Step B — POST to /media_publish with creation_id. Both calls use the same Instagram Access Token.
7
✅ Log to Google Sheets Google Sheets Node

After publishing, this node updates the original row in your content calendar. It marks Status as "Published", writes the timestamp, and saves the post ID — giving you a full audit trail.

How to configure it
Add a Google Sheets node → Operation: Update Row → match the row by Row Number → set Status = Published, Published At = {{ $now }}, Post ID = the ID returned from the platform API. Both Facebook and Instagram paths merge into this single node.
